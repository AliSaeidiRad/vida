\begin{abstract}
This report documents the experimental design, configurations, and outcomes of a multi-task convolutional neural network (CNN) developed for classification of mammographic mass regions of interest (ROIs). The network simultaneously predicts morphological and clinical attributes, including shape, margin, BI-RADS assessment, and pathology. Multiple training strategies, preprocessing pipelines, and dataset combinations were evaluated to determine an optimal configuration.
\end{abstract}

\section{Experiment 1}

\subsection*{Overview}

In the initial experiment, the network consisted of four prediction heads:
\textit{Shape}, \textit{Margin}, \textit{BI-RADS}, and \textit{Pathology}.

\section{Experiment 10 (Nominated Strategy)}
After extensive hyperparameter tuning and pipeline modifications, the following configuration was selected as the most stable and effective.

\subsection*{Configuration}

\begin{enumerate}
    \item \textbf{Backbone:} \texttt{convnext\_base.fb\_in22k\_ft\_in1k}
    
    \item \textbf{Optimizer:} \texttt{AdamW}
    
    \item \textbf{Scheduler:} \texttt{CosineAnnealingWarmRestarts}
    \begin{enumerate}
        \item Initial learning rate: $\eta_{0} = 10^{-5}$
        \item Minimum learning rate: $\eta_{\min} = 10^{-8}$
        \item Initial restart period: $T_{0} = 20$
        \item Multiplication factor: $T_{\text{mult}} = 2$
    \end{enumerate}
    
    \item \textbf{Image resizing backbone:} \textcolor{red}{Pillow}
    
    \item \textbf{Preprocess:} \textcolor{red}{ISS}
    
    \item \textbf{Datasets:} CBIS-DDSM + VIDA
\end{enumerate}

\subsection*{Observed Effects of Design Choices} The following findings were consistently observed across experiments:
in conducted experiments we find out that following changes has no effect or have negative effects:

\begin{enumerate}
    \item OpenCV-based resizing degraded performance, likely due to excessive loss of fine structural details. 
    \item Standard normalization alone was insufficient for robust learning. 
    \item Removing any prediction head reduced overall performance, indicating beneficial multi-task feature sharing. 
    \item BI-RADS predictions alone were not reliable for inferring pathology outcomes. 
\end{enumerate}

\section{Experiment 24 (Final Pipeline)}
Subsequent experiments incorporated additional datasets and an extra prediction head for malignancy probability when pathology labels were unavailable.

\subsection*{Training Strategy}

\begin{enumerate}
    \item \textbf{Stage 1 — Pretraining:} Train on CBIS-DDSM and CDD-CESM with frozen backbone. \\
    \begin{lstlisting}[language=bash, caption=Training Command based on Original Scripts]
    "$PYTHON" train.py \
    --epochs 1000 \
    --datasets cbis cdd-cesm \
    --label2id config/label2id.json \
    --preprocess iss \
    --batch-size 32 \
    --num-workers 8 \
    --lr0 1e-5 \
    --lrf 1e-7 \
    --freeze-backbone \
    --loss config/loss.json \
    --patience 100
    \end{lstlisting}
    \item \textbf{Stage 2 — Full Fine-Tuning:} Continue training with backbone unfrozen.
    \begin{lstlisting}[language=bash, caption=Training Command based on Original Scripts]
    "$PYTHON" train.py \
    --epochs 1000 \
    --datasets cbis cdd-cesm \
    --label2id config/label2id.json \
    --preprocess iss \
    --batch-size 32 \
    --num-workers 8 \
    --lr0 1e-6 \
    --lrf 1e-8 \
    --loss config/loss-optimized.json \
    --patience 100 \
    --checkpoint output/experiment-1/best_f1.pth
    \end{lstlisting}
    \item \textbf{Stage 3 — Dataset-Specific Fine-Tuning:} Fine-tune individual heads (maximum 300 epochs, early stopping patience of 30 epochs):
    \begin{lstlisting}[language=bash, caption=Training Command based on Original Scripts]
    # STAGE 3 - PART I
    "$PYTHON" train.py \
    --epochs 1000 \
    --datasets cbis cdd-cesm vida \
    --label2id config/label2id.json \
    --preprocess iss \
    --batch-size 32 \
    --num-workers 8 \
    --lr0 1e-5 \
    --lrf 1e-8 \
    --loss config/loss-optimized.json \
    --freeze-heads pathology \
    --freeze-backbone \
    --freeze-attention \
    --checkpoint output/experiment-2/best_f1.pth

    # STAGE 3 - PART II

    "$PYTHON" train.py \
    --epochs 1000 \
    --datasets cbis cdd-cesm vida vindr \
    --label2id config/label2id.json \
    --preprocess iss \
    --batch-size 32 \
    --num-workers 8 \
    --lr0 1e-5 \
    --lrf 1e-8 \
    --loss config/loss-optimized.json \
    --freeze-heads pathology shape margin \
    --freeze-backbone \
    --freeze-attention \
    --checkpoint output/experiment-3/best_f1.pth
    \end{lstlisting}
    \begin{enumerate}
    \item VIDA: Train Pathology head only; freeze attention modules and backbone.
    \item CSAW: \textcolor{red}{Excluded due to unreliable annotations.}
    \item VINDR: Train relevant heads while freezing Pathology, Margin, Shape, attention modules, and backbone.
    \end{enumerate}
\end{enumerate}

\section{Test Time Augmentation}
Overall results with TTA improved and the final pipeline for Augmentation in inference is like follow:
\begin{lstlisting}[language=Python, caption=Training Command based on Original Scripts]
from torchvision import transforms as T
import torchvision.transforms.functional as F
tta_transforms = [
    T.Lambda(lambda x: x),
    T.Lambda(lambda x: F.hflip(x)),
    T.Lambda(lambda x: F.vflip(x)),
    T.Lambda(lambda x: F.vflip(F.hflip(x))),
    T.Lambda(lambda x: F.rotate(x, 90)),
    T.Lambda(lambda x: F.rotate(x, 270)),
]
tta_weights = [
    2.0, # original
    1.5, # hflip
    1.5, # vflip
    1.0, # both flips
    0.8, # 90 degree rotation
    0.8, # 270 degree rotation
]
\end{lstlisting}
